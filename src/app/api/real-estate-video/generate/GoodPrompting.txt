MASTER SOP + PROMPT SYSTEM

One-step-at-a-time system for everything we do
This is the master operating document for the full workflow we use to go from a raw brand brief to final ad assets.
It combines:
strategy and campaign thinking
customer and angle clarity
ad format selection
scriptwriting
storyboard development
face and identity lock
character bible creation
base image prompt generation
start and end image generation
video prompt generation
voice prompt generation
image ad prompt generation
quality control
packaging and storage
The system is designed to be simple to run:
Fill → Paste → Get Output → Choose → Paste Next Prompt
Each step is designed so the user can get the next usable output in one or two actions.

HOW TO USE THIS DOCUMENT
Core Rule
Do not skip steps.
Each prompt depends on the output from the previous step.
Workflow logic
Fill the Brand Brief
Paste Prompt 1
Review the options
Choose one option
Paste the next prompt using the selected output
Keep moving until final deliverables are complete

High-detail photographic rendering, premium editorial realism, grounded in real-world camera capture.
Textures across the image must feel physically believable: skin, fabric, hair, jewelry, and background surfaces should all carry realistic material response, micro-detail, and natural imperfections. WHAT TO LEARN FROM THIS EXAMPLE
The key lesson
Do not improvise the workflow.
The logic is:
build strategy first
choose one direction
use that chosen block in the next prompt
keep the selected output clean
do not paste six conflicting options into the next step
move forward with one approved version
Operational rule
At each step, select one output and pass only that forward unless the SOP explicitly asks for multiple items.
Why this matters
This keeps the entire system:
faster
cleaner
easier to review
easier to hand off
easier to scale across projects

COMMON MISTAKES TO AVOID
1. Passing multiple options into the next step
What goes wrong
The model gets conflicting context and starts blending ideas that were supposed to stay separate.
Wrong approach
Pasting all 3 Ad DNA options into the Customer Avatar prompt.
Correct approach
Choose 1 final Ad DNA option and only pass that forward.

2. Skipping the selection step
What goes wrong
The workflow becomes messy because nothing is actually approved before moving ahead.
Wrong approach
Generating scripts before selecting the final angle or format.
Correct approach
At every stage, pause, select the strongest option, then continue.

3. Rewriting selected outputs too casually
What goes wrong
Important emotional tension, positioning, or constraints get lost between steps.
Wrong approach
Summarizing the selected avatar in your own words before pasting it into the next prompt.
Correct approach
Paste the selected output cleanly and as fully as possible using the standard handoff block.

4. Mixing UGC and cinematic logic in the same path
What goes wrong
The resulting ad becomes confused and visually inconsistent.
Wrong approach
Using a cinematic script structure and then asking for handheld creator-style UGC visuals in the same chain without a clear reason.
Correct approach
Choose one primary format and stay inside that path unless the SOP specifically asks for an alternate version.

5. Turning every step into a brainstorming session
What goes wrong
The workflow slows down and the outputs become broad, repetitive, and less usable.
Wrong approach
Asking for 10 extra variations at every stage before selecting one.
Correct approach
Use the SOP to narrow down, not expand forever.

6. Generating visuals before locking the character or product
What goes wrong
Faces drift, styling changes, and product appearance becomes inconsistent across scenes.
Wrong approach
Generating scene prompts before finalizing face-lock, product-lock, or styling rules.
Correct approach
Lock the character, product, and styling first when continuity matters.

7. Using vague prompts for later execution steps
What goes wrong
Images and videos come back generic, unstable, or visually inconsistent.
Wrong approach
Prompting with lines like “make it cinematic and premium” without defining the actual scene, subject, lighting, and framing.
Correct approach
Use the structured prompts exactly and include all locked variables.

8. Not keeping the output format clean
What goes wrong
The next prompt becomes harder to run because the handoff is messy.
Wrong approach
Mixing your own notes into the selected output block before pasting it forward.
Correct approach
Use the copy-paste connector and standard handoff format.

9. Choosing based only on what sounds pretty
What goes wrong
The team picks ideas that sound creative but are weak for actual ad performance.
Wrong approach
Selecting the most poetic concept even if it does not clearly sell the product.
Correct approach
Choose based on clarity, emotional relevance, and execution strength.

10. Treating generated output as final without QC
What goes wrong
Weak scripts, inconsistent visuals, and sloppy prompts move downstream and create bigger problems later.
Wrong approach
Using first-draft prompts or visuals without checking them.
Correct approach
Always do a quick QC pass before locking anything as final.

GOLDEN RULES FOR RUNNING THIS SYSTEM
One selected output moves forward at a time
Do not skip steps
Do not rewrite the logic casually between steps
Stay inside the selected format path
Lock continuity before scaling scenes
Use the standard handoff blocks
Review before finalizing
Keep the workflow narrow and controlled

FINAL REMINDER
This system works best when you treat it like a production pipeline, not a chat.
The model is not supposed to “figure out the project eventually.”
You are supposed to move it through the correct steps in the correct order.
Clean inputs create clean outputs.
Clean outputs make the next step fast.
That is the whole game.
